【概要】
旧 coolingshop.net / .biz の分離構成を、新しい csn 側で 1 アプリ運用へ寄せる作業を進行中。
現時点では UI よりもコードと業務フローの土台整備を優先。

【ここまでの主な変更】
1. 新サイト土台の作成
- public_html を入口にした新構成を作成
- UTF-8 前提に統一
- PHP 8 系 / MySQL 5.7 を前提にした構成へ変更
- noindex / X-Robots-Tag / robots.txt を追加

2. DB 移行の準備
- 旧バックアップ DB を umeoka_csn へ移行できるよう整形
- 旧ダンプの互換性問題を調整して phpMyAdmin でインポート完了
- .env に Sixcore の DB 接続情報を反映

3. 認証基盤の刷新
- acc テーブルを使い続ける方針で新認証を実装
- 平文 pass から password_hash へ移行できるように変更
- 初回ログイン成功時に password_hash を保存する仕組みを追加
- member_type / biz_status / password_hash を acc に追加する SQL を作成
- state=9 は削除相当としてログイン不可
- 同一メールアドレスに複数履歴がある場合、state=2 優先、次に state=1 を使う認証へ変更

4. 会員種別の整理
- 非会員: ログイン不可、購入のみ可
- 会員: 通常会員価格
- 特別会員: 特別会員価格
- 特別会員申請中: 状態に応じて docs_pending / pending / approved / rejected を扱う形へ拡張中

5. 価格表示の統一
- ProductRepository で非会員価格 / 会員価格 / 特別会員価格を計算
- 検索結果、商品詳細、カートで会員状態に応じた価格だけを表示する形に変更

6. カート統合
- .net / .biz の別カートではなく、1 つの共通カートを実装
- 検索結果と商品詳細のどちらからも同じカートへ追加可能
- カート内でも会員状態に応じた価格が再計算される
- 商品削除、数量変更の導線を追加

7. ルーティング・基本画面
- /search, /product/{id}, /cart, /login, /account, /special-member 系のルートを追加
- /check.php, /health.php を追加して疎通確認できるようにした
- トップ、検索、詳細、カート、ログイン、会員情報の基本画面を整備

8. 特別会員フローの再構築
- 旧 .biz の独立入口を廃止する方向で csn に統合中
- 現在の新フロー:
  a. /special-member/register または /special-member/apply で未ログイン申請
  b. 申請時に acc を作成し、state=1, member_type=net, biz_status=docs_pending
  c. 登録メールアドレスとパスワードでログイン
  d. /special-member/upload で名刺画像アップロード
  e. アップロード後に biz_status=pending
  f. /admin/special-member または /admin/special-members で管理承認
  g. 承認時に state=2, member_type=biz, biz_status=approved

9. 特別会員申請データ
- special_member_requests テーブルを追加
- special_member_request_files テーブルを追加
- 名刺画像は storage/uploads/special-member/{request_id} 配下へ保存

10. 管理画面の土台
- Basic 認証の簡易管理画面を追加
- /admin/special-member
- /admin/special-members
- 承認 / 却下操作を追加
- .env に ADMIN_USER / ADMIN_PASS を追加

11. メール送信の土台
- MailService を追加
- 特別会員申請受付時に本人宛メール送信
- 管理者宛通知メール送信を追加
- 送信方式は mb_send_mail() / mail() ベース
